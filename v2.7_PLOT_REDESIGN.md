## Technical Implementation

### Rate vs Range Calculation

```python
# Collect distance, MCS, and SNR data
valid_data = [(d, m, s) for d, m, s in zip(dist_vals, mcs_vals, snr_vals) 
              if d is not None and m is not None and s is not None]

# Sort by distance for smooth curve
sorted_data = sorted(valid_data, key=lambda x: x[0])
dist_sorted = [x[0] for x in sorted_data]
mcs_sorted = [x[1] for x in sorted_data]
snr_sorted = [x[2] for x in sorted_data]

# Plot both curves
line_mcs_range.set_data(dist_sorted, mcs_sorted)
line_snr_range.set_data(dist_sorted, snr_sorted)
```

**Why Sort by Distance?**
- Creates smooth, readable curves
- Shows clear degradation pattern
- Easier to identify trends
- Professional appearance

### Distance Tracking

dist_vals = []  # New list to track distance

### Interpreting Rate vs Range Plot

**What to Look For:**

1. **MCS Curve (Blue):**
   - Should start high (8-11) when close
   - Drops as distance increases
   - Sharp drops indicate coverage edge

2. **SNR Curve (Green):**
   - Should start high (>30dB) when close
   - Gradually decreases with distance
   - Below 20dB indicates poor coverage

3. **Optimal Range:**
   - Where MCS stays ≥5 and SNR ≥20dB
   - Typically 0-15m for indoor WiFi 6

4. **Coverage Edge:**
   - Where MCS drops below 5
   - Where SNR drops below 20dB
   - Indicates need for additional AP

**Example Interpretation:**
```
Distance 0-10m:  MCS 9-11, SNR 30-40dB → Excellent
Distance 10-20m: MCS 6-8,  SNR 22-30dB → Good
Distance 20-30m: MCS 3-5,  SNR 15-22dB → Fair
Distance >30m:   MCS 0-2,  SNR <15dB   → Poor (add AP)
```
