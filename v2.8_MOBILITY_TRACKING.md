---

## How It Works

### Algorithm

The system analyzes the last 3 distance measurements to determine movement direction:

1. **Distance Calculation**
   - Uses RSSI-based path loss model
   - Negative distances rounded to 0m
   - Estimated distance stored in history

2. **Trend Analysis**
   - Compares last 3 consecutive measurements
   - Calculates differences between each pair
   - Determines if all changes are in same direction

3. **State Determination**
   ```
   If all changes > +3m  → Moving Away (↑)
   If all changes < -3m  → Moving Towards (↓)
   If all changes ≤ ±3m  → Stationary (●)
   Otherwise             → Mixed (uses average)
   ```

4. **Confidence Calculation**
   - 100% if all measurements agree
   - Proportional if mixed signals
   - Based on majority direction

## Parameters

### Configurable Settings

```python
analyze_client_mobility(
    current_distance,      # Current distance in meters
    tolerance=3.0,         # Stationary threshold (±3m)
    window_size=3          # Number of measurements to analyze
)
```

**Tolerance (3.0m):**
- Changes within ±3m = Stationary
- Changes > +3m = Moving Away
- Changes < -3m = Moving Towards

**Window Size (3):**
- Analyzes last 3 measurements
- Requires 3 iterations before showing direction
- Balances responsiveness vs stability

## Use Cases

### 1. Coverage Testing
**Scenario:** Walk away from AP to find coverage edge

**What to Watch:**
- ↑ Moving Away with stable MCS = Good coverage
- ↑ Moving Away with dropping MCS = Coverage edge
- Distance where MCS drops below 5 = Coverage limit

**Example:**
```
10m: ↑ Moving Away | MCS: 11 | SNR: 40dB → Excellent
15m: ↑ Moving Away | MCS: 9  | SNR: 32dB → Good
20m: ↑ Moving Away | MCS: 7  | SNR: 25dB → Fair
25m: ↑ Moving Away | MCS: 4  | SNR: 18dB → Coverage edge!
```

### 2. Roaming Testing
**Scenario:** Walk between mesh nodes

**What to Watch:**
- ↑ Moving Away from Node A
- BSSID change (roaming event)
- ↓ Moving Towards Node B

**Example:**
```
Node A (aa:bb:cc:dd:ee:ff):
  ↑ Moving Away | Distance: 15m | RSSI: -65dBm

[ROAMING EVENT: aa:bb:cc:dd:ee:ff → 11:22:33:44:55:66]

Node B (11:22:33:44:55:66):
  ↓ Moving Towards | Distance: 8m | RSSI: -50dBm
```

### 3. Stationary Testing
**Scenario:** Test performance at fixed location

**What to Watch:**
- ● Stationary with stable metrics = Good
- ● Stationary with fluctuating metrics = Interference

**Example:**
```
● Stationary | Distance: 10m
  MCS: 9, 9, 9, 9, 9 → Stable (Good)

● Stationary | Distance: 10m
  MCS: 9, 5, 8, 4, 9 → Unstable (Interference!)
```

### 4. AP Placement Optimization
**Scenario:** Find optimal AP location

**What to Watch:**
- Walk in all directions from proposed AP location
- Note distance where ↑ Moving Away shows MCS drop
- Optimal placement = Maximum coverage radius

**Example:**
```
North: ↑ Moving Away → MCS drops at 20m
South: ↑ Moving Away → MCS drops at 25m
East:  ↑ Moving Away → MCS drops at 18m (obstruction?)
West:  ↑ Moving Away → MCS drops at 22m

Recommendation: Move AP 2m east to balance coverage
```

### Distance Calculation

```python
def estimate_distance(rssi, tx_power=-30, path_loss_exponent=3.2):
    distance = (10 ** ((tx_power - rssi) / (10 * path_loss_exponent))) - 1
    return max(0, distance)  # Ensure non-negative
```

## Limitations

### 1. Distance Estimation Accuracy
- Based on RSSI (signal strength)
- Affected by obstacles, interference
- ±2-3m accuracy typical

### 2. Tolerance Setting
- 3m tolerance may be too large for indoor
- May be too small for outdoor
- Adjustable via parameter

### 3. Window Size
- 3 measurements = ~6 seconds (2s interval)
- Slower to detect direction changes
- More stable, less noise

### 4. Initial Delay
- Requires 3 measurements before showing direction
- First 2 iterations show "Unknown"
- Normal behavior
