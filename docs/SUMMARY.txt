CRITICAL ISSUES FIXED:

1. ✅ Channel field newlines removed (was breaking CSV rows)

WHAT WAS WRONG:
--------------
User reported CSV format issues:
- Channel field contained newlines → Breaking CSV row structure
- Channel Util had "37 %" → Should be "37" (clean number)
- MCS values not validated → Need 0-12 range enforcement
- PHY mode inconsistent → Should be "11ax", "11ac", "11n"
- NSS not clean → Should be just numbers (1, 2, 3, 4)

ROOT CAUSE:
----------
The get_wifi_channel() function returns values with embedded newlines like "6g69/160\n"
This breaks CSV format because newlines are row delimiters.

SOLUTION IMPLEMENTED:
--------------------
1. MCS Validation (NEW)
   - Validates MCS values are in range 0-12
   - Invalid values show as "N/A"
   - Prevents bad data in exports

2. Channel Utilization Cleanup (IMPROVED)
   - Removes spaces: "37 %" → "37"
   - Removes % sign
   - Extracts clean number only

3. CSV-Safe Field Cleaning (CRITICAL)
   - Removes newlines (\n)
   - Removes carriage returns (\r)
   - Replaces commas with underscores
   - Strips extra whitespace
