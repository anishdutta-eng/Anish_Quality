## Issues Fixed

### 1. ✅ Folder Structure
**Problem**: KGU and DUT results were in separate top-level folders  
**Fix**: Now creates single parent folder `COMPARATIVE_<testname>` with two subfolders:
```
COMPARATIVE_ProductionTest_001/
├── KGU/
│   ├── network_diagnostics_KGU.txt
│   ├── network_diagnostics_plot_KGU.png
│   ├── diagnostics_KGU.csv
│   ├── diagnostics_KGU.json
│   └── network_report_KGU.pdf
├── DUT/
│   ├── network_diagnostics_DUT.txt
│   ├── network_diagnostics_plot_DUT.png
│   ├── diagnostics_DUT.csv
│   ├── diagnostics_DUT.json
│   └── network_report_DUT.pdf
└── comparative_report_<timestamp>.pdf
```

### 2. ✅ DUT Test Not Running
**Problem**: DUT test phase wasn't executing properly  
**Fix**: 
- Properly reset all global variables before DUT test
- Fixed exit_requested flag reset
- Ensured DUT test runs complete diagnostic suite
- Added proper folder navigation

### 3. ✅ Exit Keys
**Problem**: Both tests used 'q' key  
**Fix**:
- KGU test: Press **'d'** + Enter to end
- DUT test: Press **'q'** + Enter to end
- Updated `check_for_exit()` function to accept custom exit key

### 4. ✅ Complete Test Suite for Both Phases
**Problem**: Not all tests running for both KGU and DUT  
**Fix**: Both phases now run:
- Full diagnostic test with all iterations
- Network sanity check
- Live plotting (8 plots)
- CSV export (16 KPIs)
- JSON export (mesh topology, roaming, interference)
- PDF report generation
- All data saved in respective folders

### 5. ✅ NTF vs Wireless Issue Disposition
**Problem**: Comparison didn't clearly indicate if DUT is NTF or has issues  
**Fix**: Enhanced `compare_kgu_dut()` with intelligent disposition:

**Three Possible Outcomes:**

1. **NTF - No Trouble Found**
   - Overall status: PASS
   - Score ≥75
   - All metrics within tolerance
   - Recommendation: "Unit is acceptable for deployment"

2. **WIRELESS ISSUE DETECTED**
   - Overall status: FAIL
   - Score <75 OR critical failures
   - Specific issue types identified:
     - Throughput degradation
     - RF performance issues
     - Signal strength problems
     - Network latency issues
     - Connection stability problems
   - Recommendation: "Further investigation required. DO NOT DEPLOY."

3. **MARGINAL - Additional Testing Required**
   - Overall status: FAIL (but no critical failures)
   - Some warnings but no major issues
   - Recommendation: "Additional testing recommended before deployment"

## Code Changes

### Modified Functions

#### `check_for_exit(exit_key='q')`
```python
def check_for_exit(exit_key='q'):
    global exit_requested
    while True:
        user_input = sys.stdin.readline().strip().lower()
        if user_input == exit_key:
            exit_requested = True
            print(f"Exit requested by user (pressed '{exit_key}').")
            break
```
- Now accepts custom exit key parameter
- KGU uses 'd', DUT uses 'q'

#### `compare_kgu_dut(kgu_results, dut_results)`
Added disposition logic:
```python
comparison = {
    ...
    "disposition": "PASS",  # or "NTF - No Trouble Found" or "WIRELESS ISSUE DETECTED"
    "recommendation": ""     # Detailed recommendation text
}

# Determine final disposition
if comparison["overall_status"] == "PASS":
    comparison["disposition"] = "NTF - No Trouble Found"
    comparison["recommendation"] = "Unit is acceptable for deployment..."
else:
    # Analyze failure patterns
    failure_types = []
    if any("Throughput" in f for f in comparison["failures"]):
        failure_types.append("throughput degradation")
    if any("MCS" in f or "correlation" in f.lower() for f in comparison["failures"]):
        failure_types.append("RF performance issues")
    ...
    
    comparison["disposition"] = "WIRELESS ISSUE DETECTED"
    comparison["recommendation"] = f"Significant issues: {', '.join(failure_types)}..."
```

### Main Block Rewrite
Complete rewrite of comparative mode workflow:
1. Create parent folder `COMPARATIVE_<testname>`
2. Create KGU subfolder, run complete test
3. Generate KGU reports and save all data
4. Create DUT subfolder, run complete test
5. Generate DUT reports and save all data
6. Compare results in parent folder
7. Generate comparative report
8. Display disposition (NTF vs Wireless Issue)

## Usage

### Starting Comparative Mode
```bash
sudo python3 wl_tool12.py
```
Select mode **2**

### Workflow

#### Phase 1: KGU Test
1. Enter test name (e.g., `ProductionTest_001`)
2. Parent folder created: `COMPARATIVE_ProductionTest_001/`
3. KGU subfolder created: `COMPARATIVE_ProductionTest_001/KGU/`
4. Enter KGU AP Model and SSID
5. Enter sample interval
6. Test runs - **Press 'd' + Enter to end**
7. All KGU data exported and saved
8. KGU PDF report generated

#### Phase 2: DUT Test
1. Power OFF KGU, Power ON DUT
2. Connect to DUT
3. DUT subfolder created: `COMPARATIVE_ProductionTest_001/DUT/`
4. Enter DUT AP Model and SSID
5. Test runs - **Press 'q' + Enter to end**
6. All DUT data exported and saved
7. DUT PDF report generated

#### Phase 3: Comparison
1. Automated comparison runs
2. Results displayed in terminal
3. Comparative PDF report generated in parent folder
4. Disposition shown:
   - ✅ **NTF - No Trouble Found** (DUT is good)
   - ❌ **WIRELESS ISSUE DETECTED** (DUT has problems)
   - ⚠️ **MARGINAL** (Additional testing needed)
