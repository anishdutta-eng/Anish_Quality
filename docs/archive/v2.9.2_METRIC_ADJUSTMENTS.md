# Version 2.9.2 - Metric Adjustments

## Release Date
February 10, 2026

## Summary
Adjusted comparative testing metrics based on user feedback. Latency and roaming are now informational only and do not affect pass/fail decisions.

---

## Changes Made

### 1. ✅ Latency: Informational Only
**Reason**: Latency is server-dependent and not a reliable indicator of wireless issues

**Before**:
- Pass: ±20%
- Warning: ±30%
- Fail: >±30%
- Could cause test failure

**After**:
- Status: **INFO** (informational only)
- Threshold: "Informational only (server-dependent)"
- Never causes pass/fail
- Large deviations (>50%) shown as warnings for awareness
- Does NOT affect overall score

### 2. ✅ Roaming Events: Informational Only
**Reason**: Apple's BSSID redaction limitations prevent reliable roaming detection on macOS 14.5+

**Before**:
- Pass: ≤2 extra events
- Warning: ≤5 extra events
- Fail: >5 extra events
- Could cause test failure

**After**:
- Status: **INFO** (informational only)
- Threshold: "Informational only (Apple BSSID limitations)"
- Never causes pass/fail
- Does NOT affect overall score
- Displayed for reference only

## Updated Test Criteria

### Critical Metrics (Pass/Fail)

| Metric | ✅ Pass | ⚠️ Warn | ❌ Fail |
|--------|---------|---------|---------|
| **Peak Throughput** | ±10% | ±15% | >±15% |
| **Peak MCS** | ±1 index | ±2 index | >±2 index |
| **RSSI-MCS Correlation** | ±1.5 MCS | ±2.5 MCS | >±2.5 MCS |
| **Average RSSI** | ±3dB | ±5dB | >±5dB |

### Informational Metrics (Info Only)

| Metric | Status | Note |
|--------|--------|------|
| **Average Latency** | ℹ️ INFO | Server-dependent, not wireless issue |
| **Roaming Events** | ℹ️ INFO | Apple BSSID limitations |

## Disposition Logic Updated

### Failure Analysis Now Excludes:
- ❌ Latency deviations (server-dependent)
- ❌ Roaming event differences (unreliable on macOS)

### Failure Analysis Includes:
- ✅ Throughput degradation
- ✅ RF performance issues (MCS, correlation)
- ✅ Signal strength problems (RSSI)

### Three Possible Outcomes:

1. **NTF - No Trouble Found**
   - All critical metrics pass
   - Score ≥75
   - No wireless issues detected

2. **WIRELESS ISSUE DETECTED**
   - One or more critical metrics fail
   - Specific issue types identified:
     - Throughput degradation
     - RF performance issues
     - Signal strength problems
   - Score <75

3. **MARGINAL**
   - Some warnings but no critical failures
   - Additional testing recommended

## Technical Details

### Code Changes

#### Latency Metric
```python
# Latency is informational only - always INFO status
lat_status = "INFO"

if abs(lat_delta_pct) > 50:
    comparison["warnings"].append(f"Average Latency deviation {lat_delta_pct:+.1f}% (server-dependent, informational only)")

comparison["metrics"]["avg_latency"] = {
    "kgu": kgu_results["latency_avg"],
    "dut": dut_results["latency_avg"],
    "delta": lat_delta,
    "delta_pct": lat_delta_pct,
    "status": lat_status,
    "threshold": "Informational only (server-dependent)"
}
```

#### Roaming Metric
```python
# Roaming is informational only due to Apple's BSSID redaction
roam_status = "INFO"

comparison["metrics"]["roaming_events"] = {
    "kgu": kgu_roaming,
    "dut": dut_roaming,
    "delta": roaming_delta,
    "status": roam_status,
    "threshold": "Informational only (Apple BSSID limitations)"
}
```

#### Display Logic
```python
if status == "INFO":
    status_color = Colors.GRAY
    status_symbol = "ℹ️"
elif status == "PASS":
    status_color = Colors.GREEN
    status_symbol = "✅"
# ... etc
```

#### Disposition Logic
```python
# Analyze failure patterns to provide specific recommendations
# Note: Latency and roaming are excluded as they are informational only
failure_types = []
if any("Throughput" in f for f in comparison["failures"]):
    failure_types.append("throughput degradation")
if any("MCS" in f or "correlation" in f.lower() for f in comparison["failures"]):
    failure_types.append("RF performance issues")
if any("RSSI" in f for f in comparison["failures"]):
    failure_types.append("signal strength problems")
# Latency and roaming NOT included
```

## Rationale

### Why Latency is Informational
- **Server-dependent**: Different speedtest servers have different latencies
- **Network path**: Internet routing affects latency more than WiFi
- **Time of day**: Server load varies
- **Not wireless-specific**: Latency issues rarely indicate WiFi problems
- **False positives**: Would cause good units to fail unnecessarily

### Why Roaming is Informational
- **Apple BSSID redaction**: macOS 14.5+ redacts BSSID at kernel level
- **Unreliable detection**: Even with sudo, BSSID tracking is inconsistent
- **No workaround**: Cannot be fixed without Apple changing policy
- **False data**: Roaming counts may be inaccurate or missing
- **Not critical**: Roaming behavior varies by environment and movement
