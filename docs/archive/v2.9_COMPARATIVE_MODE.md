## Release Date
February 10, 2026

## Summary
Added production-grade **Comparative Testing Mode** for KGU (Known Good Unit) vs DUT (Device Under Test) comparison with scientific pass/fail criteria.

---

## What's New

### 🔬 Comparative Testing Mode
Complete back-to-back testing workflow:
1. **Phase 1**: Test Known Good Unit (KGU) - establish baseline
2. **Phase 2**: Test Device Under Test (DUT) - same parameters
3. **Phase 3**: Automated comparison with pass/fail decision

### 📊 Scientific Pass/Fail Criteria
Based on industry standards (TR-398, IEEE 802.11, production testing best practices):

| Metric | Pass | Warning | Fail |
|--------|------|---------|------|
| **Peak Throughput** | ±10% | ±15% | >±15% |
| **Peak MCS** | ±1 index | ±2 index | >±2 index |
| **RSSI-MCS Correlation** | ±1.5 MCS | ±2.5 MCS | >±2.5 MCS |
| **Average RSSI** | ±3dB | ±5dB | >±5dB |
| **Average Latency** | ±20% | ±30% | >±30% |
| **Roaming Events** | ≤2 extra | ≤5 extra | >5 extra |

### 🎯 Test Criteria
Three primary criteria as requested:

1. **Peak Download/Upload Speeds** (Tx Rate)
   - Compares maximum throughput achieved
   - ±10% tolerance (industry standard)

2. **Peak MCS vs RSSI**
   - Compares maximum modulation/coding scheme
   - ±1 MCS index tolerance

3. **RSSI Ramp Down to MCS Ramp Down**
   - Analyzes rate vs range degradation curves
   - Ensures both units degrade similarly as distance increases
   - Correlation analysis with ±1.5 MCS tolerance

### 📄 Comparative Report
Professional PDF report includes:
- Side-by-side comparison table (KGU vs DUT)
- Overall PASS/FAIL status with score (0-100)
- Metric-by-metric analysis with thresholds
- Critical failures, warnings, passed criteria
- Test disposition with recommendations
- Root cause analysis for failures

### 🔧 New Functions

#### `store_test_results(test_type)`
Captures and stores complete test results including:
- All statistics (RSSI, SNR, MCS, Tx Rate, Latency, Distance)
- Peak values
- Connection quality metrics
- Raw data for detailed analysis

#### `calculate_rssi_mcs_correlation(rssi_values, mcs_values)`
Analyzes RSSI-to-MCS degradation curve:
- Groups data into RSSI buckets (5dB intervals)
- Calculates average MCS per bucket
- Returns correlation data for comparison

#### `compare_kgu_dut(kgu_results, dut_results)`
Comprehensive comparison engine:
- Evaluates all metrics against scientific thresholds
- Calculates overall score (0-100)
- Identifies failures, warnings, passed criteria
- Returns detailed comparison dictionary

#### `generate_comparative_report(kgu_results, dut_results, comparison)`
Generates professional PDF report:
- Test information for both units
- Comparison table with all metrics
- Color-coded pass/fail status
- Disposition and recommendations

## Usage

### Starting Comparative Mode
```bash
sudo python3 wl_tool12.py
```

Select option **2** for Comparative Test mode.

### Workflow
1. **Connect to KGU** (ensure it's the ONLY router on)
2. **Run KGU test** - tool stores baseline
3. **Power OFF KGU, Power ON DUT**
4. **Connect to DUT**
5. **Run DUT test** - same parameters as KGU
6. **Automated comparison** - instant pass/fail decision
7. **Generate report** - comprehensive PDF

### Important Rules
- ⚠️ **Only ONE router powered on at a time**
- Use **identical configuration** on both routers
- Run tests in **same location** (don't move laptop)
- Use **same test duration** for both

## Technical Details

### Tolerance Justification

**Throughput (±10%)**:
- Industry standard for RF production testing
- Accounts for environmental variation and measurement uncertainty
- Based on TR-398 WiFi performance testing standards

**RSSI (±3dB)**:
- Standard RF measurement tolerance
- Typical antenna/RF chain variation
- Aligns with IEEE 802.11 specifications

**MCS (±1 index)**:
- Allows minor RF variations
- Catches significant modulation issues
- Based on WiFi 6 MCS table analysis

**Latency (±20%)**:
- Network latency naturally varies
- Accounts for internet path differences
- Large deviations indicate real problems

**RSSI-MCS Correlation (±1.5 MCS)**:
- Tests fundamental RF performance curve
- Ensures rate vs range behavior matches
- Critical for detecting antenna/calibration issues

### Scoring System
- Start: 100 points
- Critical failure (throughput, MCS, correlation): -25 points
- Major failure (RSSI, latency): -15 points
- Minor failure (roaming): -15 points
- Warning: -5 to -10 points
- **Pass threshold**: ≥75 points AND no critical failures
