#### All KPIs Included

- **BSSID** - Access point MAC address (mesh node identifier!)
- **RSSI** - Signal strength in dBm
- **SNR** - Signal-to-Noise Ratio in dB
- **Noise** - Noise floor in dBm
- **Tx Rate** - Transmit rate in Mbps
- **Latency** - Ping latency in ms
- **MCS Index** - Modulation and Coding Scheme (0-11)
- **PHY Mode** - 802.11 standard (ac, ax, etc.)
- **NSS** - Number of Spatial Streams
- **Channel Utilization** - Congestion percentage
- **Distance** - Estimated distance from AP in meters
- **Health Status** - Excellent/Good/Bad

### 2. 🔓 SSID Redaction Fix (macOS 14.5+)

**Your Request:** "Can you search the internet a workaround to how to get the SSID of the wifi network in macbook since wdutil shows it as redacted"

**Problem:** macOS 14.5+ returns `<redacted>` for SSID due to privacy restrictions

**Solution:** Multiple fallback methods implemented!

#### Method 1: networksetup (Primary)
```bash
networksetup -getairportnetwork en0
```
Works on macOS 14.5+ and bypasses privacy restrictions.

#### Method 2: ipconfig getsummary (Fallback)
```bash
ipconfig getsummary en0 | grep 'SSID :'
```
Works on some macOS versions.

#### Method 3: system_profiler (Legacy)
```bash
system_profiler SPAirPortDataType
```
Older method, still works on some systems.

#### Method 4: CoreWLAN (Python)
```python
import CoreWLAN
iface = CoreWLAN.CWInterface.interface()
ssid = iface.ssid()
```
Direct API access when available.

#### Automatic Fallback
The tool tries all methods automatically and uses the first one that works!

```python
def get_ssid():
    # Try Method 1: networksetup
    # Try Method 2: ipconfig
    # Try Method 3: system_profiler
    # Try Method 4: CoreWLAN
    # Return best result
```

#### If Still Redacted
If all methods fail, you'll see:
```
SSID: SSID Not Available (Privacy Restricted)
```

But don't worry - all other diagnostics still work perfectly!

### 3. 🌐 Mesh Node Detection & Tracking

**Your Request:** "Also give me options to know which mesh node am I connected to? Is it possible to distinguish from the client if it connects to a different mesh node but of the same wifi network?"

**Answer:** YES! Using BSSID tracking (already implemented, now enhanced!)

#### How Mesh Detection Works

**BSSID = Unique identifier for each mesh node/AP**

Even if all nodes share the same SSID (network name), each mesh node has a unique BSSID (MAC address). When you roam between nodes, the BSSID changes!

#### What You'll See

**During Test (Every 10 Iterations):**
```

  Current node (BSSID): aa:bb:cc:dd:ee:ff

  Recent transitions:
    11:22:33 → aa:bb:cc at 45.2s
    aa:bb:cc → dd:ee:ff at 67.8s
    dd:ee:ff → aa:bb:cc at 89.3s
```

## Mesh Network FAQ

### Q: How do I know if I have a mesh network?
**A:** If you see multiple unique BSSIDs in the CSV or "Mesh network detected" in the output, you have a mesh network!

### Q: Can I name my mesh nodes?
**A:** Yes! Note the BSSID when you're at each location, then create a mapping:
```
aa:bb:cc:dd:ee:ff = "Living Room"
11:22:33:44:55:66 = "Bedroom"
```

### Q: Why does BSSID change but SSID stays the same?
**A:** That's how mesh networks work! All nodes share the same SSID (network name) but each has a unique BSSID (MAC address).

### Q: How can I tell which node is the main router?
**A:** The node you connect to first (when closest to router) is usually the main one. Check the first BSSID in your CSV.

### Q: Is frequent roaming bad?
**A:** Not necessarily! It means your mesh is working. But if roaming happens too often (>10 times in 5 minutes), it might indicate:
- Overlapping coverage (nodes too close)
- Weak signals (nodes too far)
- Aggressive roaming settings

### Q: Can I force connection to a specific node?
**A:** Not directly from this tool, but you can:
1. Identify the node's BSSID
2. Move closer to that node
3. The system will naturally prefer it
